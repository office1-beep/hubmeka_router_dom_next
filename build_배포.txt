
가. Next.js에서 빌드 결과물이 생성되는 위치는 어떤 명령어를 사용했느냐와 
어떤 렌더링 방식을 선택했느냐에 따라 달라집니다

	1. .next 폴더 (기본 빌드) : 서버 기반 빌드

		ㄱ. 서버 사이드 렌더링(SSR)을 위한 로직
		ㄴ. 정적 페이지
		ㄷ. 그리고 최적화된 JavaScript 파일들이 들어 있습니다
		ㄹ. 개발 모드(.next/static)

		a. server/: 서버에서 실행될 페이지와 API 노드 코드
		b. static/: 브라우저가 내려받을 JS, CSS 파일 (청크 파일)
		c. cache/: 빌드 속도를 높이기 위한 캐시 데이터

	2. out 폴더 (정적 내보내기) : 정적 호스팅 빌드
		: Next.js를 Static Site Generation (SSG) 방식으로 설정해서 
		  순수 HTML/CSS/JS 파일만 뽑아내고 싶을 때 사용합니다
		ㄱ. 설정 방법: next.config.js 파일에 output: 'export' 설정을 추가하고 빌드합니다.
		ㄴ. 프로젝트 루트의 out/
		ㄷ. Nginx, Apache, S3, GitHub Pages 등 
		    별도의 Node.js 서버 없이 정적 호스팅을 할 때 사용합니다.

나. 서버에 배포해야 할 파일은 **"어떤 방식으로 서비스를 운영할 것인가"**에 따라 명확히 갈립니다. 
   가장 일반적인 두 가지 상황으로 나누어 정확한 위치를 집어드릴게요.

	1. Node.js 서버를 직접 운영할 때 (SSR/ISR 사용)

	2. 정적 호스팅을 할 때 (SSG 전용)

	3. 도커/최적화형
